NAME

s16_root - evaluate root  

SYNOPSIS

#include "s16math.h"
qm_n s16_root(qm_n a, qm_n n);
 

DESCRIPTION

Evaluate a^(1/n) and return principal root.

Fractional operands are evaluated by applying successive exponential identities to simplify them into a set of integer valued power and root operations of acceptable degree.

The resolution of fractional exponents is constrained by using a reduction table to obtain manageable values for power/root calculations. This approximates the fractional part of the index to the nearest 1/16'th value when binary resolution is used and the nearest 1/10'th value when decimal resolution is used.

Accuracy is good for radicands with integer valued indices and for fractional indices that reduce to integer multiples of 1/2. e.g. 0.5, 4.5. Fractional indices less than one that reduce to integer multiples of 1/8 - e.g. 0.125, 0.625 - produce accurate roots for indices in the range 0.25 - 4.00 inclusive.

Fractional indices greater than 0 but less than 0.25 cause inaccurate results, as do indices that reduce to any odd multiple of 1/16.  

PARAMETERS

Parameters are supplied as 16 bit integers having an implied binary point with 7 bits of resolution, or an implied decimal point with two digits of resolution.
a
number to find root of i.e. radicand.
n
root to be found - i.e. index.
 

RETURN

The root is returned as a 16 bit integer having an implied binary point with 7 bits of resolution, or an implied decimal point with two digits of resolution.

If an even root of a negative number is requested, the function returns zero. Otherwise the function returns a negative root [8].  

NOTES

Internal 32 bit intermediate results.

Calculators vary in handling roots of negative numbers. The funcs.bc package [8] issues an error message followed by a zero when an even integer valued root of a negative number is requested. The Debian calculate-gtk desktop calculator package returns a complex number. This function follows funcs.bc practice.

For odd integer valued roots of negative numbers, both of those packages return the negative of the root of the corresponding positive number, as does this function.

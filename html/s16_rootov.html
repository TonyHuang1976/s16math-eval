NAME

s16_rootov - Check for nth root overflow.  

SYNOPSIS

#include "s16math.h"
qm_n s16_rootov(qm_n a, qm_n b, qm_n c);
 

DESCRIPTION

The relationship a=b^(1/c) is checked for consistency. Overflow is found if
•
the index is zero (because x^(1/0) divides by zero), or
•
the index is negative and the radicand is zero (because the reciprocal of zero to a fractional power also divides by zero), or
•
the index is also odd and radicand and root differ in sign, or
•
a positive index is even and either the radicand or the root is negative (see notes)

The function does not check for the case where the index is one and the root and radicand are not equal because s16_root() treats an index of one as a corner case and always returns a root equal to the radicand.  

PARAMETERS

a
root
b
radicand
c
index

Parameters are supplied as 16 bit integers having implied binary points with 7 bits of resolution, or implied decimal points with two digits of resolution.  

RETURN

Zero if no overflow. Non-zero if overflow.  

NOTES

Does not catch all cases. Negative indices can produce root reciprocals smaller than the minimum resolution of the library, meaning that a root with a value of zero does not always mean overflow has occurred.

Numbers with imaginary or complex roots are deemed to be overflow.

The function s16_root() returns zero to indicate overflow if called with a negative radicand and an even index.

NAME

s16_mulov - Check for multiply overflow  

SYNOPSIS

#include "s16math.h"
s16 s16_mulov(qm_n a, qm_n b, qm_n c);
 

DESCRIPTION

Divide or multiply overflow occurs when the result is positive and the operands are of different signs, or if the result is negative and the operands are of the same sign. Otherwise there is no overflow, but see NOTE.

In the case where n != 0, n*0 is never deemed to be overflow.  

PARAMETERS

a
product.
b
multiplicand.
c
multiplier.
 

RETURN

Zero if no overflow. Non-zero if overflow.  

NOTE

A value of zero for the product when the multiplicand and multiplier are both non- zero is not by itself considered an error, since the condition may occur due to rounding.

s16_mulov() is defined as s16_divov().

NAME

s16_powov - Check for nth power overflow  

SYNOPSIS

#include "s16math.h" qm_n s16_powov(qm_n a, qm_n b, qm_n c);
 

DESCRIPTION

The relationship a=b^c is checked for consistency.

Overflow is deemed to have occurred if a negative power has a positive base, if a negative power has a negative base with an even exponent, or if a positive power has a negative base with an odd exponent. Otherwise, there is no overflow.  

PARAMETERS

a
power
b
base
c
exponent

Parameters are supplied as 16 bit integers having implied binary points with 7 bits of resolution, or implied decimal points with two digits of resolution.  

RETURN

Zero if no overflow. Non-zero if overflow.  

NOTES

Probably does not catch all cases.
